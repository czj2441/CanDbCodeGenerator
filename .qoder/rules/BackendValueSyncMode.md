# 后端值同步模式 (Backend Value Sync Mode)

## 核心原则
前端 UI 状态与后端数据保持一致时，必须严格遵循**后端权威**与**全局状态单一驱动**原则。禁止在前端进行乐观更新的手动回滚，禁止使用前端缓存的旧值进行错误恢复，所有状态变更的最终确认值必须显式来自后端响应，杜绝任何前端状态猜测与静默推断。

## 具体规则

### 1. 输入值处理
- UI 控件通过双向绑定直接写入全局状态。
- 用户交互时，控件值与全局状态同步变化，然后才向后端发送变更请求。
- **禁止**在前端保存旧值。
- **禁止**做乐观更新后的手动回滚。

### 2. 成功路径
- 后端接受变更后，广播一个状态事件，事件中必须携带变更后的字段值。
- 前端收到该事件时，用事件中的值覆盖全局状态。
- 此时全局状态值与控件值一致，UI 无需额外操作。

### 3. 失败路径
- 后端拒绝变更时，必须在错误响应的附加信息中返回当前正确的字段值（即后端权威值）。
- 前端在错误处理中从附加信息取出该值，直接写入全局状态。
- 由于全局状态值被改回后端确认的值，双向绑定自动将 UI 控件同步到正确状态。
- 同时弹出提示告知用户错误信息。

### 4. 关键约束
- **单一驱动源**：全局状态是唯一驱动源，UI 控件不独立维护状态。
- **禁止使用旧值恢复**：错误恢复时，写入全局状态的值必须来自后端响应，严禁使用前端缓存的旧值（遵循“禁止静默推断”原则，不猜测状态）。
- **后端响应规范**：后端必须在错误响应中携带当前正确的字段值，不能只返回错误码和错误信息。
- **成功确认规范**：成功路径的确认值也必须来自后端事件，不依赖请求发送时的本地值。
